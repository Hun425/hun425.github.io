REP-Engine

개인 프로젝트

개요

기존 배치 기반 추천 시스템의 한계(실시간성 부족)를 극복하기 위해 설계한 이벤트 기반 실시간 추천 엔진입니다. 초당 수천 건의 유저 행동 로그를 처리하여 200ms 이내에 추천 결과를 제공합니다.

아키텍처 및 기술적 의사결정

  • Event Pipeline: Kafka + Avro (Schema Registry) 기반의 이벤트 드리븐 아키텍처
  • Concurrency: Coroutines + Virtual Thread를 도입하여 블로킹 I/O 상황에서도 높은 처리량 확보
  • Storage Strategy: Redis(Hot/Vector Cache)와 Elasticsearch(Vector Search) 하이브리드 전략

트러블 슈팅

Problem: 실시간 벡터 갱신 시 Lost Update 발생

수십만 유저의 행동 로그가 동시에 들어올 때, 취향 벡터 갱신 과정에서 동시성 문제 발생.

Solution: 유저별 Mutex Lock 도입

Caffeine 캐시 기반의 Mutex Lock을 적용하여 동일 유저 이벤트를 직렬화하고, Redis를 활용해 읽기 성능 최적화.

Result: 데이터 정합성 100% 보장 및 P99 응답 속도 5ms 미만 달성
